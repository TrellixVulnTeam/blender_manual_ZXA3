-- Baking Smoke Simulations TODO2.8: .. figure:: /images/physics_smoke_baking_interface.png :align: right *Smoke Cache* options. Baking is used to store the outcome of a simulation so it does not need to be recalculated. Smoke baking settings are in Properties ‣ Physics ‣ Smoke ‣ Cache . Unlike most physics simulations smoke physics has some settings that are specific to smoke.
Định Dạng Tập Tin -- File Format File format that the cache data is to be stored.
Bộ Nhớ Đệm Điểm -- Point Cache Blender's own caching format.
OpenVDB Advanced and efficient storage method developed by DreamWorks Animation . The simulation fields can be found in the .vdb files under the following names: "color" "density" "heat" "heat old" (the temperature at the previous frame) "flame" "fuel" "react" (reaction coordinates, used for fire) "velocity" "shadow" (the shadows of the volume computed for viewport rendering) "texture coordinates" (used for turbulence) Nén -- Compression Method of data compression. Zip Efficient but slower compression method. Blosc Multi-threaded compression with about the same quality and size as Zip . Không Có -- None Do not compress the data. Độ Lớn của Dữ Liệu -- Data Depth Bit depth for writing all scalar (including vectors), lower values reduce the file size of the cache. Float (Nửa) -- Float (Half) Half float (16 bit data). Gives less data with the benefit of smaller file sizes. Float (Đơn) -- Float (Full) Full float (32 bit data). Gives more data at the cost of larger file sizes. For other options see the General Baking docs for more information. Baking can only been done once your blend-file is saved. If your blend-file has not been saved, the Smoke Cache panel will be disabled.
