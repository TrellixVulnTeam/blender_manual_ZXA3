Xóa và Áp Dụng -- Clear & Apply
Xóa Sạch -- Clear Tham Chiếu -- Reference Chế Độ -- Mode Chế Độ Vật Thể -- Object Mode Trình Đơn -- Menu Vật Thể (Object) ‣ Xóa Sạch (Clear) ‣ Xóa Sạch Vị Trí/Xóa Sạch Tỷ Lệ/Xóa Sạch Xoay Chiều/Xóa Sạch Tọa Độ Gốc (Clear Location / Clear Scale / Clear Rotation / Clear Origin) Phím Tắt -- Hotkey Alt-G , Alt-S , Alt-R Clearing transforms simply resets the transform values. The objects location and rotation values return to 0, and the scale returns to 1. Xóa Vị Trí -- Clear Location Alt-G Clear (reset) the location of the selection. This will move the selection back to the coordinates (0, 0, 0). Xóa Tỷ Lệ -- Clear Scale Alt-S Clear (reset) the scale of the selection. This will resize the selection back to the size it was when created. Xóa Sự Xoay Chiều -- Clear Rotation Alt-R Clear (reset) the rotation of the selection. This will set the rotation of the selection to 0 degrees in each plane. Clear Origin Clears (resets) the offset of the child objects origin. This will cause child objects to move to the origin of the parent.
Tùy Chọn -- Options Xóa Delta -- Clear Delta Clear the delta transform in addition to clearing the primary transforms. (Appears in the Adjust Last Operation panel.)
Áp Dụng -- Apply These operations lets you apply several transformations to the selected objects. The object transformation coordinates are transferred to the object data. If the objects have hierarchical descendants, it also applies those transformations to their children.
Áp Dụng các Biến Hóa của Vật Thể -- Apply Object Transformations Tham Chiếu -- Reference Chế Độ -- Mode Chế Độ Vật Thể -- Object Mode Trình Đơn -- Menu Vật Thể (Object) ‣ Áp Dụng (Apply) ‣ Vị Trí / Xoay Chiều / Tỷ Lệ / Xoay Chiều & Tỷ Lệ (Location / Rotation / Scale / Rotation & Scale) Phím Tắt -- Hotkey Ctrl-A Applying transform values essentially resets the values of object's location, rotation or scale, while visually keeping the object data in-place. The object origin point is moved to the global origin, the rotation is cleared and scale values are set to 1. For simple cases you won't notice any difference the 3D View or rendered output, however modifiers and constraints may depend on object transformation. Armature Objects While applying transformations to armatures is supported, this does not apply to their pose location, animation curves or constraints. This tool should be used before rigging and animation. When running Apply Transform , the Adjust Last Operation panel lets you choose the combination of transformations to apply.
Tùy Chọn -- Options Vị Trí -- Location Apply (set) the location of the selection. This will make Blender consider the current location to be equivalent to 0 in each plane i.e. the selection will not move, the current location will be considered to be the "default location". The Object origin will be set to actual (0, 0, 0) (where the colored axis lines intersect in each view). Xoay Chiều -- Rotation Apply (set) the rotation of the selection. This will make Blender consider the current rotation to be equivalent to 0 degrees in each plane i.e. the selection will not rotated, the current rotation will be considered to be the "default rotation". Tỷ Lệ -- Scale Apply (set) the scale of the selection. This will make Blender consider the current scale to be equivalent to 0 in each plane i.e. the selection will not scaled, the current scale will be considered to be the "default scale". Xoay Chiều và Tỷ Lệ -- Rotation and Scale Apply (set) the rotation and scale of the selection. Do the above two applications simultaneously. Áp Dụng các Tính Chất -- Apply Properties Modify properties such as curve vertex radius, font size and bone envelope according to the applied transformation.
Biến Hóa sang Delta -- Transforms to Deltas Tham Chiếu -- Reference Chế Độ -- Mode Chế Độ Vật Thể -- Object Mode Trình Đơn -- Menu :menuselection: ` Vật Thể (Object) --> Áp Dụng (Apply) --> ` Converts primary object transformations to delta transforms , any existing delta transforms will be included as well. Location to Deltas Rotation to Deltas Scale to Deltas Tất Cả các Biến Hóa sang Delta -- All Transforms to Deltas Converts all primary transformations to delta transforms. Biến Hóa đã Hoạt Họa thành Deltas -- Animated Transform to Deltas Converts the primary transformation animations (animations done to the translation, scale, and, rotation values) to delta transforms.
Tùy Chọn -- Options Hoàn Lại các Giá Trị -- Reset Values Clear primary transform values after transferring to deltas.
Trực Quan Hóa sự Biến Hóa -- Visual Transform Tham Chiếu -- Reference Chế Độ -- Mode Chế Độ Vật Thể -- Object Mode Trình Đơn -- Menu Vật Thể (Object) ‣ Áp Dụng (Apply) ‣ Trực Quan Hóa sự Biến Hóa (Visual Transform) Phím Tắt -- Hotkey Ctrl-A Apply (set) the result of a constraint and apply this back to the Object's location, rotation and scale.
Trực Quan Hóa Hình Học như Khung Lưới -- Visual Geometry as Mesh Tham Chiếu -- Reference Chế Độ -- Mode Chế Độ Vật Thể -- Object Mode Trình Đơn -- Menu Object ‣ Apply ‣ Visual Geometry to Mesh Phím Tắt -- Hotkey Ctrl-A Apply the visual state of all selected objects (modifiers, shape keys, hooks, etc.) to object data. This is a way to freeze all object data into static meshes, as well as converts non-mesh types to mesh. For details, see the Chuyển Đổi Sang -- Convert To mesh.
Make Instances Real Tham Chiếu -- Reference Chế Độ -- Mode Chế Độ Vật Thể -- Object Mode Trình Đơn -- Menu Object ‣ Apply ‣ Make Instances Real Phím Tắt -- Hotkey Shift-Ctrl-A Make Instances Real creates a new object for each instance generated by the selected ones, and removes any direct instancing from those. In the end, each instance becomes a real object. This applies to both direct (from verts or faces...) and indirect (from particle system...) instancing. In case you have tens of thousands of instances (from particles for example), this can significantly slow down Blender, which does not always deal well with that many objects in a scene.
Tùy Chọn -- Options By default, new objects will be added to the same collection as the one containing their instancer, without keeping any hierarchy relationships. This behavior can be altered with the following options. Phụ Huynh -- Parent If Keep Hierarchy is not set, parents all the generated objects to the former instancer. Otherwise, parents all the generated objects which are not already parented to their respective instancer, or its matching new copy (this is important in case of recursive instancing, see the note below). Duy Trì Thứ Bậc -- Keep Hierarchy Preserves internal hierarchies (i.e. parent relationships) in the newly generated objects. Usually, to get a new hierarchy as close as possible from the instancing one, you'll want to enable both of these options. Preserving relationships in recursive instancing cases (instancers instancing other instancer objects, etc.) is only supported to some extent currently. Simple cases (like an empty instancing a collection containing instances of some other collections) will usually work, but more complex cases will fail to fully reproduce the whole instancing hierarchy.
